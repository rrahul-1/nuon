001 — Runner Health Check

Periodic runner-level health check signal that aggregates process state into runner status. Click + on any code line to leave an inline comment.

Problem

Runner health is monitored at the process level (process_healthcheck, every 5-15 min) but there is no periodic runner-level check. The existing offline_check uses a 24-hour threshold and is admin-triggered only. A runner can have zero active processes and still appear "active" indefinitely.

Architecture

Requirements

  • Signal fires every 15 minutes via cron emitter on runner-signals queue
  • Org runners: check for active org process → active or offline
  • Install runners: check for active install process (primary) → active or offline. Check for mng process → metadata warning if absent
  • Skip runners in transitional states
  • Emit runner.health_check counter with per-check tags
  • Emit Datadog event on active → offline transitions with full owner context
