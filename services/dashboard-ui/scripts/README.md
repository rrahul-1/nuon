# Dashboard UI Scripts

## Scripts

### `clean-dist.js`
Removes the `dist/` directory, recreates `dist/assets/`, and copies `client/index.html` into `dist/`. Runs at the start of both `bun run dev` and `bun run build` to ensure a clean output directory. Uses Node `fs` for cross-platform compatibility.

### `build-css.js`
Processes `client/styles.css` through PostCSS + Tailwind CSS and outputs to `dist/assets/styles.css`. Supports `--watch` mode for development.

### `hash-assets.js`
Post-build step that rewrites `index.html` references to point at the content-hashed filenames generated by the bundler, and content-hashes the Tailwind CSS output. Runs automatically as part of `bun run build`.

### `compress-assets.js`
Pre-compresses all `.js` and `.css` files in `dist/assets/` with gzip (level 9), generating `.gz` files alongside the originals. The Go BFF serves the `.gz` variant when the client sends `Accept-Encoding: gzip`. Only runs as part of `bun run build` (not dev).

### `generate-api-types.js`
Generates TypeScript types from the Nuon OpenAPI spec into `client/types/nuon-oapi-v3.d.ts` using `openapi-typescript`. Runs automatically before `dev` and `tsc`.

### `dev-server.js`
Bun-based dev proxy that sits in front of the Go BFF. Proxies all requests to the BFF, injects an SSE live-reload script into HTML responses, and watches `dist/` for changes to trigger reloads.

### `dev.sh`
Entry point used by nctl to start the dashboard locally. Builds the Go BFF server, starts it in the background, waits for it to write a port file, then runs `bun run dev` with live reload.

## Build pipeline

**Dev** (`bun run dev`):
```
clean-dist.js → parallel(build-css.js --watch, bun build --watch, dev-server.js, generate-api-types)
```

**Production** (`bun run build`):
```
clean-dist.js → parallel(bun build --splitting --minify, build-css.js) → hash-assets.js → compress-assets.js
```

## Usage

### Generate API types (production API, default)
```bash
bun run generate-api-types
```

### Generate API types from local API
```bash
NUON_API_URL=http://localhost:8081 bun run generate-api-types
```

### Generate API types from a local spec file
```bash
NUON_OPENAPI_SPEC_FILE=./path/to/spec.json bun run generate-api-types
```

## Environment Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `NUON_API_URL` | `https://api.nuon.co` | API URL to fetch the OpenAPI spec from |
| `NUON_OPENAPI_SPEC_FILE` | — | Local spec file path (takes precedence over `NUON_API_URL`) |
| `HTTP_PORT` | `4000` | Port for the Go BFF (dev server proxies from port + 1) |
