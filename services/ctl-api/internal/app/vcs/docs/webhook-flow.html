VCS Webhook Event Flow

GitHub Push/PR → GithubEvent → VCS Connection Signal (per org) → App Branch Run

1
GitHub sends webhook
HTTP
Endpoint: POST /v1/vcs/webhooks/:subscription_id/events
Headers: X-GitHub-Event: push   X-Hub-Signature-256: sha256=...   X-GitHub-Hook-Installation-Target-ID: 12345
Body: Raw JSON payload (ref, repository.full_name, commits, etc.)
Auth: HMAC-SHA256 signature verified against VCSWebhookSubscription.WebhookSecret
signature verified
2
Store GithubEvent
DB S3 write_webhook_event.go
Lookup: VCSWebhookSubscription by subscription_id path param
Extract: github_install_id from X-GitHub-Hook-Installation-Target-ID header (fallback: subscription's value)
Blob upload: Raw body → blobstore.Blob → S3 at blobs/github_events/{blob_id}
// GithubEvent record created in DB:
ID: "ghevt..."
GithubInstallID: "12345"
EventType: "push"
Payload: *blobstore.Blob{s3_key: "blobs/github_events/blob_..."}
Status: "success"
event stored
3
Batch-create VCSConnectionEvents
DB write_webhook_event.go
Query: SELECT * FROM vcs_connections WHERE github_install_id = '12345'
Why multiple: Same GitHub App installation can be connected by multiple Nuon orgs
Per connection: Create a VCSConnectionEvent record with FK to the GithubEvent, scoped to conn.OrgID
// VCSConnectionEvent record (per connection):
ID: "vcsconnevt..."
VCSConnectionID: "vcsconn..."
GithubEventID: "ghevt..." // FK
OrgID: "org..."
Status: "queued"
FOR EACH VCSConnectionEvent created
4
Enqueue github_event signal per VCSConnectionEvent
SIGNAL write_webhook_event.go
Queue lookup: queueClient.GetQueueByOwner(conn.ID, "vcs_connections")
Enqueue: queueClient.EnqueueSignal() with github_event signal
// Signal payload enqueued:
type: "github_event"
vcs_connection_event_id: "vcsconnevt..."
HTTP returns 200 — async from here
5
Signal Execute: load event chain + parse payload
TEMPORAL signals/github_event/execute.go
Activity: GetVCSConnectionEvent — loads VCSConnectionEvent, follows FK to GithubEvent, downloads payload from S3
Parse: parsePushEvent(payload) → extracts repo (owner/name) and branch from ref
Context: All routing info comes from the VCSConnectionEvent record (connection ID, org ID, github event FK)
6
Find matching App Branches
TEMPORAL DB activities/find_matching_app_branches.go
Activity: FindMatchingAppBranches
Query: connected_github_vcs_configs WHERE vcs_connection_id AND repo = "owner/name" AND branch = "main"
JOIN app_branch_configs (latest config per app branch, deduped)
Returns: List of {AppBranchID, AppBranchConfigID} matches
FOR EACH matching App Branch
7
Enqueue vcs-push signal to App Branch queue
SIGNAL signals/github_event/execute.go
Activity: EnqueueSignalToOwner — resolves queue by (AppBranchID, "app_branches")
// vcs-push signal payload:
type: "vcs-push"
app_branch_id: "appbr..."
app_branch_config_id: "appbrcfg..."
event_type: "push" // or "pull_request"
plan_only: false // true for PRs
head_sha: "" // set for PRs
pr_number: null // set for PRs
8
App Branch Run triggered
TEMPORAL signals/branches/vcspush/signal.go
Signal type: vcs-push
Execute: TriggerAppBranchRunFromVCSPush activity — creates an App Branch Run with the push/PR metadata
Result: New App Branch Run created, which orchestrates component builds for the branch

Key Models

VCSWebhookSubscription Per GitHub App installation. Holds webhook secret for HMAC verification. Created when VCS connection is set up.
GithubEvent Raw webhook event. Payload stored in S3 via blobstore. Not org-scoped (shared across connections with same install ID).
VCSConnection Org-scoped link to a GitHub App installation. Matched by github_install_id. Has its own signal queue.
VCSConnectionEvent Junction: links a GithubEvent to a specific VCSConnection + OrgID. Created during signal execution.
ConnectedGithubVCSConfig Links a VCSConnection to an AppBranchConfig with repo + branch matching rules.
AppBranchRun The final output: a run that builds/deploys components for the matched branch.

File Reference

vcs/service/write_webhook_event.go HTTP handler: verify signature, store event, fan out signals per VCS connection
vcs/signals/github_event/signal.go Signal struct with GithubEventID, VCSConnectionID, OrgID
vcs/signals/github_event/execute.go Signal execution: parse payload, create VCSConnectionEvent, find matching app branches, enqueue vcs-push
vcs/signals/github_event/parse.go Parse push/PR payloads: extract repo, branch, PR metadata
vcs/worker/activities/get_github_event.go Temporal activity: load GithubEvent + download payload from S3
vcs/worker/activities/find_matching_app_branches.go Temporal activity: query ConnectedGithubVCSConfigs JOIN AppBranchConfigs
apps/signals/branches/vcspush/signal.go vcs-push signal: triggers TriggerAppBranchRunFromVCSPush activity
